🛠️ Methodology

  1. Data sources. Per-tool usage APIs (request counts, token counts, model names) + org reporting graph from the company directory.
  2. Roll-up. Each person's raw usage joined with their reporting depth (d1–d4 below me) and their direct manager.
  3. Grouping. Renders 7 tabs (Me / Senior Leader+ Network / Office PM Benchmark / Senior Leader Group / Team / Full Team / Licensed Tools Coverage), each a filtered leaderboard.
  4. Status buckets. Power user / Steady / Light / Inactive — based on weekly request count thresholds.
  5. Streaks. Consecutive weeks of activity, surfaced as a 🔥 badge to drive friendly competition.
  6. Publishing. A scheduled agent regenerates this HTML daily, drops it into a shared location, and pings the team channel on material rank changes.
Why it works

Visibility + light gamification surface the natural champions, who then mentor lighter users. Adoption rose from ~40% to ~78% over the iteration cycle.

What I'd change

Add per-tool quality signals (not just volume) — high-token, low-value usage shouldn't beat high-leverage agent runs.
