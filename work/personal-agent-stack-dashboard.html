Personal Agent Stack

🐾 Pattern · Adopt · Compound

Six personal agents — each small, each opinionated, each composable. Together they cover the operational overhead that used to eat ~5 hrs a week.

🤖 6 agents 📅 Refreshed weekly 🧩 Composable, not monolithic

Operating principles

🧩

Small & specialized

Each agent owns one job and does it well. Anything bigger becomes a composition, not a monolith.

🪢

Shared schema

Every agent reads/writes the same memory + identity model so they can hand work off without re-explaining context.

🛡️

Humans in the loop

Drafts, not sends. Suggestions, not actions. Trust is earned per task type, never granted blanket.

The agents

📅

meeting-room-agent

🟢 Stable

Books meeting rooms, manages recurring series, respects scheduling preferences and onboarding constraints.

Triggers: calendar invite hooks · Outputs: room bookings, draft updates

📊

ai-usage-tracker

🟢 Stable

Scheduled agent that pulls AI tool usage data daily, rolls it up by reporting depth, and publishes the team dashboard.

Triggers: daily cron · Outputs: HTML dashboard, weekly digest

📖

storyteller

🟡 Beta

Transforms raw notes, data, and ideas into polished product narratives. Structures thinking into crisp, audience-aware decks & one-pagers.

Triggers: manual · Outputs: HTML decks, markdown docs

🪢

task-wrangler

🟡 Beta

Reads emails, chat threads, and meeting notes; extracts action items; routes them to the right system (todo, ticket, follow-up email draft).

Triggers: inbox / chat scans · Outputs: structured action items

🏗️

org-change-agent

🔒 Local only

Tracks org changes (reorgs, scope transfers, attrition) against a reusable 7-workstream checklist. Surfaces automation potential.

Triggers: manual · Outputs: per-reorg checklists, automation scoring

🐑

stack-shepherd

🔒 Local only

Meta-agent. Watches the other agents (uptime, last-success, drift) and surfaces issues before I notice them in practice.

Triggers: hourly cron · Outputs: health digest, alerts
