DRY architecture

One generic runner.py driven by environment variables. Add a new scheduled job by dropping a prompt file + a launchd plist or workflow — no new Python. 

scheduled-agent/
├── runner.py                   ← reads AGENT_NAME, PROMPT_FILE, OUTPUT_NAME
├── prompts/
│   ├── daily-digest.txt        ← daily-task-agent instructions
│   └── weekly-ai-usage.txt     ← ai-usage-tracker instructions
├── launchd/
│   ├── com.Sauravi.daily-digest.plist        ← daily 11:00 IST
│   └── com.Sauravi.weekly-ai-usage.plist     ← Mondays 09:00 IST
├── .github/workflows/
│   ├── daily-agent.yml         ← cloud (manual until M365 service auth)
│   └── weekly-ai-usage.yml     ← Mondays 03:30 UTC = 09:00 IST  ⭐
├── the cloud platform/                       ← Kubernetes path (Dockerfile, CronJob, Vault)

Optional: run daily-task-agent daily on your Mac (lid closed, AC plugged)

daily-task-agent needs your local Outlook / Jira / Confluence auth, so it stays on the Mac. Combine launchd with pmset wake and your laptop self-wakes for the 11:00 IST job. 

cd ~/workspace/scheduled-agent
mkdir -p logs output

cp launchd/com.Sauravi.daily-digest.plist     ~/Library/LaunchAgents/
cp launchd/com.Sauravi.weekly-ai-usage.plist  ~/Library/LaunchAgents/

launchctl load ~/Library/LaunchAgents/com.Sauravi.daily-digest.plist
launchctl load ~/Library/LaunchAgents/com.Sauravi.weekly-ai-usage.plist

# Wake the Mac every day at 10:55 IST so the 11:00 job actually fires
sudo pmset repeat wakeorpoweron MTWRFSU 10:55:00
pmset -g sched

⚡ Caveat: Mac must be on AC power for wake-from-sleep. Battery-only = no wake. If the Mac was off entirely, wakeorpoweron will boot it (supported Macs only).

Executive summary

  • Schedule daily-task-agent & the AI usage report on your Mac via launchd + pmset wake. 5-minute setup, works today.
  • If you outgrow the Mac path, move to the internal CI/CD platform — requires moving the repo to an APM-tagged Walmart org first. The .the CI platform is ready.
  • Do not bother with GHE Actions on personal repos. It's disabled at the platform level and will return 404 forever.
  • the cloud platform only when scheduling for a team — 2-7 days of IAM work for one person's report is overkill.
